이론 1: 썸네일 바이럴 (추정 70%)
실제 확산 경로:

1. 누군가 끝까지 봄
2. "이거 감동적이다" → 공유/태그
3. 다른 사람들은 썸네일을 먼저 봄
4. 썸네일 = 장작더미 소 or 우는 소
5. "뭐지? 클릭해봐야지"
6. 이미 호기심이 있는 상태로 시청
→ 초반 후킹 대신 썸네일이 후킹 역할

이론 2: 알고리즘 행운 (20%)
가능한 시나리오:

1. 초기 소수가 끝까지 시청 (완성도 높아서)
2. 평균 시청 시간 = 15초 (거의 100%)
3. 알고리즘: "오! 이거 좋은 영상이네"
4. 더 많은 사람에게 노출
5. 눈덩이 효과
이론 3: 2차 확산 효과 (10%)
바이럴 메커니즘:

플랫폼 A (예: 틱톡) → 낮은 조회수
   ↓
누군가 다른 플랫폼에 재업로드
   ↓
플랫폼 B (예: 인스타) → 폭발
   ↓
역으로 플랫폼 A도 상승
⚠️ 이건 위험한 구조입니다
왜 위험한가?
1. 재현 가능성 낮음
같은 퀄리티로 100개 만들면
→ 90개는 조회수 낮음
→ 10개만 운이 좋으면 터짐
2. 초반 이탈률 높음
[일반 시청자 행동]
0-2초: "별로네" → 스크롤 (70%)
2-5초: "음... 볼까?" (20%)
5초 이후: "오! 재밌네" (10% 남음)
3. 알고리즘 불리
유튜브/틱톡 알고리즘 = 초반 3초 중요
초반 이탈 많음 = 추천 감소

📊 심리학적 원리
1. 인지 일치 효과 (Cognitive Consistency)
기대 = 실제
→ 만족감 + 신뢰
→ 계속 시청

기대 ≠ 실제  
→ 배신감 + 불신
→ 이탈
2. 즉각 보상 원리 (Instant Gratification)
클릭 이유 = "저 장면 보고 싶어"
0초에 보여줌 = 즉각 보상
→ 도파민 분비
→ "더 보고 싶어" 심리
3. 정보 갭 이론 (Information Gap Theory)
[0초] 충격적 장면 (일치)
   ↓ "만족"
[1초] "하지만 어떻게?"
   ↓ "궁금해!"
[플래시백] 설명 시작
   ↓ "알고 싶어!"
→ 끝까지 시청

2-A. 시점 변화
원본: 3인칭 관찰자 시점

변형안:

1. 1인칭 시점 (소의 눈으로 보기)
   - 주관적 카메라
   - 관객이 소가 됨
   - 몰입도 ↑↑

2. 다중 시점
   - 소 시점 → 원숭이 시점 → 경찰 시점
   - 입체적 이해

3. 반전 시점
   - 처음엔 원숭이 시점으로 시작
   - "장난치는 게 재밌어!"
   - 중간에 소 시점으로 전환
   - "아... 이게 피해자 입장이구나"
2-B. 시간 구조 변형
원본: 선형 시간 (순차 진행)

변형안:

1. 역순 구조
   [시작] 감옥 원숭이
   [중간] 어떻게 여기까지?
   [끝] 처음 나무 베던 장면
   → "아, 그래서 그랬구나"

2. 루프 구조
   [시작] 소가 다시 나무 베기
   [중간] 전체 스토리
   [끝] 다시 시작점
   → "또 시작이야..." (무한 노동 은유)

3. 병렬 구조
   [화면 분할]
   좌: 소의 하루
   우: 원숭이들의 하루
   → 대비 효과 극대화
2-C. 결말 변주
원본: 정의 구현 (경찰 체포)

변형 가능:

1. 해피엔딩 강화
   - 경찰이 소에게 메달 수여
   - 소가 새 집 얻음
   - 마을 사람들이 응원

2. 씁쓸한 결말
   - 원숭이 체포되지만
   - 소는 여전히 나무 베고 있음
   - "정의는 이뤄졌지만..."

3. 반전 결말
   - 사실 원숭이도 배고파서 그랬음
   - 소와 원숭이 화해
   - 함께 일하게 됨

4. 열린 결말
   - 원숭이 탈출 암시
   - "이야기는 계속된다"
   - 시즌2 기대감

5. 복수 결말
   - 소가 직접 해결
   - 초능력 획득
   - 원숭이들 혼내줌
